■ 一、文档概述
本文档旨在为代客外汇交易系统的客户画像与产品推荐功能提供一套完整的业务规则体系。通过整合客户交易行为（RFM）、资产规模（AUM）、风险等级（KYC）及敞口/行业特征四大维度，构建精准的客户分群模型，并据此匹配最适合的外汇交易产品。

文档覆盖以下九大类交易产品：
1. 即期（Spot）
2. 远期（Forward）
3. 掉期（Swap）
4. 普通期权（Vanilla Option）
5. 加强即期（Enhanced Spot）
6. 加强远期（Enhanced Forward）
7. 比率远期（Ratio Forward）
8. 价优远期（Best Rate Forward）
9. 区间远期（Range Forward）

■ 二、客户画像体系
客户画像由以下四大维度构成，各维度协同作用，形成完整的客户视图。
▸ 2.1 RFM 行为标签
RFM 是衡量客户交易行为的经典模型，包含三个核心指标：
指标
含义
业务解释
R（Recency）
最近一次交易时间
距离客户最近一笔外汇交易的天数。R 越小，客户越活跃。
F（Frequency）
交易频率
一定周期内（通常12个月）的交易笔数。F 越高，客户粘性越强。
M（Monetary）
交易金额
一定周期内交易总金额（折算UCNY）。M 越大，客户贡献度越高。

RFM 分箱规则：
• R 分箱：≤7天=5分，≤30天=4分，≤90天=3分，≤180天=2分，>180天=1分
• F 分箱：按 NTILE(5) 五等分
• M 分箱：按 NTILE(5) 五等分
▸ 2.2 AUM 资产标签
资产规模反映客户的资金实力，是判断适合推荐产品层级的重要依据。

▸ 2.3 风险等级（KYC）
基于 KYC 问卷评定，是产品适当性管理的核心依据。
风险等级
可推荐产品范围
保守型（C）
仅适合即期、加强即期、掉期等低风险产品
平衡型（B）
适合远期、价优远期、普通期权（买入）等中风险产品
激进型（A）
适合全部产品，含比率远期、区间远期等复杂结构
▸ 2.4 敞口/行业特征
识别客户的真实贸易背景和汇率敞口，是精准推荐的关键。
• 应收外币账款：出口型企业，有结汇需求，关注汇率贬值风险
• 应付外币货款：进口型企业，有购汇需求，关注汇率升值风险
• 季节性敞口：特定行业（如农产品、纺织）在固定周期有集中收付汇
• 双向现金流：制造贸易企业既有收汇也有付汇，适合掉期和比率远期
• 无贸易背景：个人客户或投资型客户，适合即期、外币理财及期权

■ 五、核心推荐规则矩阵
基于四大画像维度和九大类产品，按客户分群制定精准推荐规则。
▸ 5.1 场景一：高价值活跃套保型企业
画像特征：RFM 高（R<30天，F>10，M>7000万元），AUM 充足，风险平衡/激进，有真实贸易敞口。
推荐目标：锁定成本/收益，管理汇率波动。
推荐产品
触发条件（细分场景）
远期 / 价优远期
客户有敞口预测，直接推荐对应期限的远期锁定
区间远期
客户认为当前汇率已接近年内高点/低点，希望锁定底线同时保留优化空间
加强远期
客户希望比普通远期拿到更好的价格，愿意承担一定的灵活性限制
掉期
客户有短期外币贷款还款压力，或需调整外币存款期限结构
▸ 5.2 场景二：中低频/稳健型企业或个人高净值
画像特征：RFM 中等（F 较低，M 中等），AUM 高，风险保守/平衡，可能有季节性需求。
推荐目标：简单易用，避免过度复杂。
推荐产品
触发条件（细分场景）
即期
当前有即时支付需求，或客户对汇率走势不确定选择观望
加强即期
客户暂时不想做远期，但希望汇率向有利方向波动时能获得更优价格
普通期权（买入）
客户极度担心汇率大幅波动，但不想占用全额本金做远期，愿意支付期权费买保险
▸ 5.3 场景三：进出口制造/贸易行业
画像特征：行业标签为 Manufacturing/Trading，有持续双向现金流（既有收汇也有付汇）。
推荐目标：管理双向敞口，优化资金效率。
推荐产品
触发条件（细分场景）
比率远期
客户对汇率波动有一定容忍度，通过卖出虚值期权覆盖买入期权成本（零成本策略）
掉期
管理账期错配，如收到美元货款但3个月后才需支付，期间可做掉期增值
远期
有明确的单笔大额应收/应付敞口时，使用标准远期锁定
▸ 5.4 场景四：休眠唤醒/低价值客户
画像特征：RFM 低（R>180天），AUM 低，风险未知/保守。
推荐目标：激活交易，降低门槛。
推荐产品
触发条件（细分场景）
即期
仅推荐基础功能，当汇率出现大幅波动新闻时推送提醒并附带即期换汇入口
加强即期
作为比普通即期更有吸引力的「钩子」产品，引导客户体验差异化服务

■ 六、推荐理由话术模板
推荐产品时需向客户/客户经理展示推荐原因，增强信任感与采纳率。以下为各产品的标准化推荐理由模板：
推荐产品
推荐理由文案示例
远期
根据您的贸易背景，贵司 Q3 预计有 200 万美元付款需求，当前 USD/CNY 远期价格处于年内高位，建议锁定购汇成本。
加强即期
您目前暂无紧急付款安排，但人民币近期波动较大。建议设置「加强即期」，若汇率触及 7.20，自动享受优于市价的结汇。
区间远期
您希望锁定最低结汇价 7.05，同时如果能涨到 7.15 则愿意部分成交。区间远期可在不支付期权费的情况下满足此需求。
比率远期
通过组合策略，您无需支付期权费即可获得比普通远期更优的结汇价格，前提是需承担双倍交割义务。
普通期权
当前汇率隐含波动率处于高位，支付少量期权费即可获得汇率保险，若汇率向有利方向波动仍可全额受益。
掉期
您的美元贷款将于 3 个月后到期，同时有闲置美元存款，建议通过掉期交易匹配资金期限。
加强即期
汇率市场波动加剧，设置加强即期可在即期交易基础上，捕捉更有利的汇率点位。

■ 七、附录：推荐引擎伪代码
以下伪代码展示推荐引擎的核心逻辑流程，供开发团队参考实现。
附录 A：推荐引擎核心逻辑（伪代码）
function recommend_products(customer):
    // Step 1: 初始化产品池
    available_products = all_products
 
    // Step 2: 硬过滤（合规检查）
    if customer.risk_level == "Conservative":
        remove [普通期权, 加强远期, 比率远期, 区间远期]
    if not customer.has_derivative_license:
        remove [远期, 掉期, 加强远期, 比率远期, 区间远期]
    if customer.AUM < 100_000:
        remove [比率远期, 普通期权]
 
    // Step 3: 精准匹配（业务规则）
    // 场景一：高价值套保企业
    if customer.is_high_value_corp and customer.exposure > 0:
        if customer.exposure_type == "Payable":
            recommend 远期, 价优远期
            if market_volatility_high():
                recommend 区间远期
        elif customer.exposure_type == "Receivable":
            recommend 远期, 加强远期
 
    // 场景二：稳健型/个人高净值
    elif customer.AUM > 500_000 and customer.rfm.freq < 5:
        recommend 加强即期, 即期
 
    // 场景三：进出口制造/贸易
    elif customer.industry in ["Manufacturing", "Trading"]:
        recommend 比率远期, 掉期, 远期
 
    // 场景四：休眠/低价值
    elif customer.rfm.recency > 180:
        recommend 即期, 加强即期
 
    // 兜底推荐
    if no recommendations:
        recommend 即期
 
    return top 3 recommendations
