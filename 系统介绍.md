# 银行代客外汇交易系统 — 系统介绍

## 一、系统概述

**银行代客外汇交易系统（xfunds）** 是一套面向银行代客外汇业务的前后端分离系统，覆盖即期、远期、掉期、期权四大外汇产品的完整交易生命周期。系统实现从牌价展示、交易录入、存续期管理、到期处理，到审批复核、保证金管理、客户与机构权限管理的全流程数字化，并满足总行/分行/支行三级机构的 Maker-Checker（经办-复核）工作流与监管报送要求。

- **后端服务**：Spring Boot 3.2.5 + MyBatis + MySQL，端口 8080
- **前端工程**：Vue 3 + Vite + Element Plus + Pinia，端口 5173
- **数据库**：MySQL 8.0+，库名 `xfunds`，字符集 utf8mb4
- **默认登录账号**：admin / admin123

## 四、核心业务流程

### 4.1 交易状态机

交易状态（TradeStatus）覆盖完整生命周期：

```
DRAFT(草稿) → PENDING_CHECK(待复核) → ACTIVE(生效) → MATURED(到期) → SETTLED(已交割)
                                      ↓
                              REJECTED(已拒绝) → 可修改重提
```

特殊状态：SETTLE_FAILED(交割失败)、DEFAULTED(违约)、CLOSED(已平仓)、ROLLED_OVER(已展期)、EARLY_SETTLED(提前交割)、EARLY_DEFAULTED(提前违约)、EXERCISED(已行权)、ABANDONED(已放弃)、PREMIUM_SETTLED(期权费已结清)。

### 4.2 Maker-Checker 工作流

- 经办员提交交易 → 状态由 DRAFT 流转至 PENDING_CHECK，生成复核待办
- 复核员不能与经办员为同一人
- **复核通过** → ACTIVE（生效）；超本机构审批额度则升级到上级机构授权
- **复核拒绝** → REJECTED，可修改后重新提交
- **退回经办** → 生成 MODIFY 任务回退给原经办人
- 角色权限：admin 可跨机构；非 admin 复核人员仅能处理本机构任务

### 4.3 任务类型（TaskType）

CHECK(复核)、AUTHORIZE(授权)、CHECK_LIFECYCLE(生命周期复核)、MARGIN_CALL(追保)、MATURITY_REMIND(到期提醒)、EXERCISE_REMIND(行权提醒)、QUOTE_CHECK(报价复核)、MODIFY(修改交易)、EARLY_DEFAULT(提前违约)、EARLY_DELIVERY(提前交割)、ROLLOVER_ORIGINAL(原价展期)、ROLLOVER_MARKET(市价展期)。

特殊生命周期任务（提前交割/原价展期/市价展期）采用审批制：提交时仅组装数据并创建任务，审批通过后才生成实际的掉期交易（自动 ACTIVE，无需二次复核）。

### 4.4 牌价机制

- 汇率精简为 4 个字段：totalBuyRate(总行买入价)、totalSellRate(总行卖出价)、branchCustomerBuyRate(分/客买价=客户卖出价，低值)、branchCustomerSellRate(分/客卖价=客户买入价，高值)
- 牌价定时任务每小时刷新，满足银行利润约束：totalBuyRate > branchCustomerBuyRate，totalSellRate < branchCustomerSellRate
- 汇率精确到小数点后 4 位
- 牌价历史保留最长 1 周后自动清理

### 4.5 保证金管理

- 交易审批通过时扣减保证金（余额减少）
- 交割成功时退还保证金（余额增加）
- 交割失败时没收保证金（余额减少）
- 轧差交易：提前违约提交时冻结轧差金额，交割时扣减并释放冻结，拒绝/退回时释放冻结
- 提前违约需校验轧差账户余额 ≥ 轧差金额

## 五、数据库设计

数据库共 **24 张表**，采用主表 + 子表的分表设计，主表 `fx_trade_master` 统一索引，按 `trade_type` 路由到对应子表。

### 5.1 机构与权限（4 表）
- `fx_org`：机构表（总行/分行/支行三级，含审批额度）
- `fx_role`：角色表
- `fx_user`：用户表
- `fx_user_role`：用户角色关联表

### 5.2 客户相关（3 表）
- `fx_customer`：客户表（对公/个人）
- `fx_customer_account`：客户账户表（区分现汇 SPOT/现钞 CASH，含冻结金额）
- `fx_customer_balance`：客户余额表

### 5.3 报价（1 表）
- `fx_quote`：报价表（SPOT/FORWARD/SWAP 三类，含期限、到期日、4 档汇率、远期点、掉期点）

### 5.4 交易主表与子表（5 表）
- `fx_trade_master`：交易主表（业务编号、类型、状态、客户、币种、金额、方向、起息/到期日、交割方式、经办/复核/授权人、监管报送字段、轧差字段）
- `fx_spot_trade`：即期交易子表
- `fx_forward_trade`：远期交易子表（含展期/提前交割/提前违约标志）
- `fx_swap_trade`：掉期交易子表（近端/远端两腿完整字段）
- `fx_option_trade`：期权交易子表（欧式/美式、执行价、期权费、观察期、行权/放弃标志）

### 5.5 保证金与授信（4 表）
- `fx_margin_account`：保证金账户表
- `fx_margin_txn`：保证金交易流水表
- `fx_credit_limit`：客户授信额度表
- `fx_margin_param`：保证金计算参数表

### 5.6 生命周期与审批（2 表）
- `fx_trade_lifecycle`：交易生命周期事件表（关联 fx_user 取真实姓名）
- `fx_approval_log`：审批日志表

### 5.7 任务与调度（3 表）
- `fx_task`：待办任务表（含认领人、角色、机构、业务负载）
- `fx_scheduled_job_detail`：定时任务详情表
- `fx_scheduled_job_log`：定时任务执行日志表

### 5.8 系统参数（2 表）
- `fx_sys_param`：系统参数表
- `fx_option_param`：期权参数表

### 5.9 监管报送预留

交易主表预留 RCPMIS 报送标志、结售汇用途代码（purpose_code）、外汇用途编码（fx_purpose_code）、上报时间等字段。

## 九、系统特色

1. **完整产品覆盖**：即期、远期、掉期、期权四大外汇产品全生命周期管理
2. **三级机构 + Maker-Checker**：总行/分行/支行三级机构，经办-复核-授权工作流，审批额度自动升级
3. **复杂生命周期操作**：提前交割、提前违约、原价/市价展期、全部违约、保证金增补，自动生成关联交易
4. **期权业务完备**：欧式/美式期权、价内提醒、行权/放弃审批制、期权费交割
5. **保证金与轧差**：保证金扣减/退还/没收、轧差账户冻结/释放机制
6. **定时任务驱动**：到期交割与牌价刷新自动化，支持失败重试
7. **监管报送预留**：RCPMIS、结售汇用途代码等字段预留
8. **统一待办队列**：跨产品任务统一调度，支持认领、超时升级、机构权限隔离
